## Domain Overview

The User Management domain handles user lifecycle operations: registration, authentication, profile management, and role-based access control.

## Architecture

```
Client App
  │
  ├── POST /api/users/register     → user-svc
  ├── POST /api/auth/login          → auth-svc → token-svc
  ├── GET  /api/users/:id           → user-svc
  ├── PUT  /api/users/:id/profile   → user-svc → storage-svc
  └── GET  /api/users/:id/roles     → auth-svc → permission-svc
```

## Endpoint Summary

| Method | Path | Service | Pattern |
|--------|------|---------|---------|
| POST | `/api/users/register` | user-svc | direct |
| POST | `/api/auth/login` | auth-svc | fan-out |
| POST | `/api/auth/refresh` | auth-svc | direct |
| GET | `/api/users/:id` | user-svc | direct |
| PUT | `/api/users/:id/profile` | user-svc | fan-out |
| DELETE | `/api/users/:id` | user-svc | saga |
| GET | `/api/users/:id/roles` | auth-svc | proxy |

## Key Concepts

- **JWT Authentication**: Access tokens (15min) + refresh tokens (7d) stored in httpOnly cookies
- **Role hierarchy**: superadmin > admin > editor > viewer
- **Soft delete**: Users are never physically deleted; `deletedAt` timestamp is set
