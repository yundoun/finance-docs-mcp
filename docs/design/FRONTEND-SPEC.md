## 1. 사용자 이해

### 사용자 계층

| 계층 | 특징 | 사이트 이용 패턴 | 디자인 요구 |
|------|------|---------------|-----------|
| **재테크 입문자** (20~30대) | 첫 투자, 용어 모름 | 검색으로 진입 → 단일 문서 읽기 → 관련 문서 탐색 | 쉬운 언어, 큰 글씨, 시각적 비교표, 학습 경로 안내 |
| **경험 투자자** (30~40대) | 기본은 아는데 제도/세금 확인 필요 | 특정 주제 검색 → 정확한 수치 확인 → 나감 | 빠른 검색, 정확한 수치 강조, 기준 시점 명확 |
| **AI 개발자** | MCP로 연결해서 쓰는 사람 | 사이트 방문 안 할 수 있음 (MCP 전용) | API 문서, .mcp.json 설정 안내 |
| **콘텐츠 크리에이터** | 블로그/유튜브용 팩트체크 | 검색 → 출처 확인 → 인용 | 출처 표기 명확, 공유 기능 |

### 핵심 사용 시나리오

```
시나리오 1: "ISA가 뭐야?"
  Google/Naver 검색 → 사이트 진입 → ISA 문서 → 
  비교표에서 핵심 파악 → 관련 문서(IRP, ETF) 탐색

시나리오 2: "ETF 세금 얼마야?"
  사이트 내 검색(⌘K) → ETF 세금 섹션 직접 이동 → 
  수치 확인 → 나감

시나리오 3: "30대인데 뭐부터 해야 해?"
  챗봇에 질문 → RAG가 ISA + ETF + 자산배분 조합 → 
  단계별 안내 → 각 문서 링크 클릭
```

## 2. 디자인 방향성

### 레퍼런스 분석

| 사이트 | 가져올 것 | 안 가져올 것 |
|--------|----------|------------|
| **Stripe Docs** | 깔끔한 타이포그래피, 여백, 코드 블록 스타일 | 개발자 전용 톤 |
| **Vercel Docs** | 사이드바 네비게이션, 다크모드, 검색 UX | 기술 중심 구조 |
| **토스 디자인** | 금융을 쉽게 전달하는 일러스트/아이콘, 한국어 최적화 | 앱 중심 UI |
| **KB Think** | 금융 용어 사전 구조, 카테고리 분류 | 올드한 UI, 느린 로딩 |
| **금감원 e-금융교육** | 교육 콘텐츠 구조, 난이도 표시 | 관공서 디자인 |
| **Investopedia** | 용어별 독립 페이지, SEO 구조, 관련 문서 연결 | 광고 과다 |

### 디자인 원칙 5가지

1. **신뢰감 (Trust)** — 금융 사이트는 신뢰가 최우선. 깨끗한 레이아웃, 출처 명시, 면책 조항 가시성
2. **접근성 (Accessibility)** — 재테크 입문자도 압도당하지 않는 정보 밀도. 점진적 공개(Progressive Disclosure)
3. **검색 중심 (Search-First)** — 대부분의 사용자는 검색으로 진입. ⌘K 검색 + 챗봇이 핵심 UX
4. **학습 경로 (Learning Path)** — 개별 문서가 아닌 연결된 학습 흐름. prerequisite → 현재 → related
5. **모바일 우선 (Mobile-First)** — 재테크 관심층은 모바일 비중 높음. 터치 타깃 충분, 읽기 편한 레이아웃

### 톤 & 무드

```
키워드: 신뢰, 깔끔, 현대적, 교육적, 접근 가능
컬러: 블루 계열 (금융 신뢰) + 민트/그린 (성장/수익) + 화이트 (깨끗함)
안티패턴: 화려한 그라데이션, 주식 차트 배경, 돈/금화 일러스트
```

## 3. 정보 구조 (IA)

### 페이지 구조

```
/                          ← 랜딩 페이지
├── /docs/{locale}/{path}  ← 문서 페이지 (핵심)
├── /chat                  ← AI 챗봇 페이지
├── /search                ← 검색 결과 페이지 (선택적)
└── /about                 ← 프로젝트 소개 + MCP 연결 안내
```

### 문서 페이지 레이아웃

```
┌──────────────────────────────────────────────────────────┐
│  Header: 로고 + 검색(⌘K) + 언어(KO/EN) + 테마 + GitHub  │
├──────────┬───────────────────────────────────┬───────────┤
│          │                                   │           │
│ Sidebar  │      Document Content             │   TOC     │
│          │                                   │ (Table of │
│ 카테고리  │  난이도 배지                        │ Contents) │
│ 트리     │  제목                              │           │
│          │  한 줄 요약 (하이라이트 박스)         │ ## 개요   │
│ ◉ 기초   │                                   │ ## 상세   │
│   주식   │  본문                              │ ## 실전   │
│   ETF    │  비교표                            │ ## FAQ    │
│   ...    │  계산 예시                          │           │
│ ◉ 계좌   │                                   │           │
│   ISA    │  관련 문서 카드                      │           │
│   IRP    │  면책 조항                          │           │
│ ◉ 세금   │                                   │           │
│ ◉ 전략   │                                   │           │
│          │                                   │           │
└──────────┴───────────────────────────────────┴───────────┘
                        Footer
```

### 랜딩 페이지 구조

```
┌─────────────────────────────────────────────┐
│  Hero: "AI에게 물어볼 수 있는 금융 교과서"      │

└─────────────────────────────────────────────┘
```

## 4. 컴포넌트 설계

### 핵심 컴포넌트

| 컴포넌트 | 역할 | 우선순위 |
|---------|------|:-------:|
| **SearchModal** | ⌘K 전역 검색. BM25 결과 실시간 표시 | P0 |
| **ChatPage** | RAG 챗봇. SSE 스트리밍 응답 | P0 |
| **DocPage** | MDX 렌더링 + TOC + 메타정보 | P0 |
| **Sidebar** | 카테고리 트리 네비게이션 | P0 |
| **DifficultyBadge** | beginner/intermediate/advanced 시각 표시 | P1 |
| **ComparisonTable** | 비교표 전용 스타일 (강조, 하이라이트) | P1 |
| **RelatedCards** | 관련 문서 카드 (문서 하단) | P1 |
| **DisclaimerBanner** | 면책 조항 고정 표시 | P0 |
| **LanguageSwitch** | KO ↔ EN 전환 | P1 |
| **FormulaBlock** | 계산식 렌더링 (PER = 주가/EPS 등) | P2 |

### 난이도 배지 디자인

```
[beginner]     → 초록 배지, "입문" 라벨
[intermediate] → 파랑 배지, "중급" 라벨  
[advanced]     → 보라 배지, "심화" 라벨
```

### 한 줄 요약 하이라이트 박스

문서 최상단에 핵심 정의를 눈에 띄게 표시:

```
┌─ 💡 한 줄 요약 ──────────────────────────────────────────┐
│  주식은 기업의 소유권 지분을 의미하며, 주식을 사면 그 기업의 │
│  주주가 되어 경영 참여 권리와 이익 배분 권리를 갖는다.       │
└──────────────────────────────────────────────────────────┘
```

## 5. 기술 스택

| 기술 | 선택 | 근거 |
|------|------|------|
| **프레임워크** | Next.js 15+ (App Router) | SSR/SSG, SEO, Vercel 배포 최적, 한/영 라우팅 |
| **스타일링** | Tailwind CSS 4 + shadcn/ui | 빠른 개발, 일관된 디자인 시스템, 다크모드 내장 |
| **MDX 렌더링** | next-mdx-remote 또는 contentlayer | 빌드 시 MDX → 페이지 생성 |
| **검색** | MCP 서버 REST API (/api/search) | 이미 구축된 BM25 엔진 활용 |
| **챗봇** | MCP 서버 SSE (/chat) | 이미 구축된 RAG 파이프라인 |
| **상태 관리** | Zustand (최소) | 테마, 검색 모달, 사이드바 토글 |
| **배포** | Vercel (무료 티어 → 도메인) | OQ-2 결정 |
| **폰트** | Pretendard (한글) + Inter (영문) | 가독성, 무료, 웹 최적화 |
| **아이콘** | Lucide React | 경량, 일관된 스타일 |

### coolstay-docs에서 기존 Next.js가 아닌 React Router

coolstay-docs는 React 19 + React Router 7 + Vite로 구현되었습니다. finance-docs-mcp는 **SEO가 중요**하므로 (Google/Naver 검색 유입) Next.js App Router로 새로 구축하는 것이 적합합니다.

| 항목 | coolstay-docs (기존) | finance-docs-mcp (새) |
|------|---------------------|----------------------|
| 프레임워크 | React Router 7 + Vite | **Next.js 15 App Router** |
| 렌더링 | CSR (SPA) | **SSG + SSR** |
| SEO | 약함 (SPA) | **강함** (정적 생성) |
| 라우팅 | 파일 기반 (Vite glob) | **App Router** (/ko/docs/...) |
| 배포 | Vercel (SPA 리라이트) | **Vercel (네이티브)** |

## 8. 반응형 브레이크포인트

```
Mobile:   < 768px   — 사이드바 숨김, 풀 너비 콘텐츠, 햄버거 메뉴
Tablet:   768~1024  — 사이드바 접힘, TOC 숨김
Desktop:  > 1024    — 사이드바 + 콘텐츠 + TOC 3단 레이아웃
Wide:     > 1440    — 최대 너비 제한 (콘텐츠 800px)
```

## 9. Open Design Questions

1. **로고**: 텍스트 로고 vs 아이콘 + 텍스트? 금융 느낌의 아이콘?
