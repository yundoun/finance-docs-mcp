# 문서 품질 체크리스트

문서 작성 후, PR 제출 전 이 체크리스트를 확인합니다.
리뷰어도 이 기준으로 리뷰합니다.

---

## 1. Frontmatter

- [ ] `title` — 주제를 명확히 설명하는 한글명 (영어 약어 있으면 병기)
- [ ] `description` — 검색 스니펫에 노출될 한 줄 설명 (50~100자)
- [ ] `category` — `hub`, `concept`, `account`, `strategy`, `tax`, `market`, `glossary`, `update`, `scenario` 중 하나
- [ ] `tags` — 관련 키워드 3~7개 (검색 정확도에 영향)
- [ ] `difficulty` — `beginner`, `intermediate`, `advanced` 중 하나
- [ ] `related` — 관련 문서 경로 3개 이상
- [ ] `updated` — `YYYY-MM-DD` 형식의 작성/수정일

## 2. 정확성

- [ ] 모든 수치에 **기준 시점** 명시 ("2026년 6월 기준")
- [ ] 수치의 **공식 출처** 명시 (금융위, 국세청, 한국은행, 금감원, 한국거래소, 법제처)
- [ ] **특정 종목/상품/증권사 이름** 없음
- [ ] **수익률 보장 표현** 없음
- [ ] 법령/제도 명칭이 **정식 명칭**과 일치

## 3. 구조

- [ ] **한 줄 요약** 섹션 있음 (## 한 줄 요약)
- [ ] **개요** 섹션 있음
- [ ] **상세** 섹션 있음 (비교표, 수치, 구조 설명)
- [ ] **실전 적용** 섹션 있음 (시나리오, 계산 예시, 주의사항)
- [ ] **자주 묻는 질문** 섹션 있음 (3~5개)
- [ ] **관련 문서** 섹션 있음
- [ ] **면책 조항** 문서 하단에 있음

## 4. 연결성

- [ ] `related` 3개 이상
- [ ] 본문에서 다른 개념 언급 시 해당 문서 링크 포함
- [ ] `prerequisite`가 필요한 경우 명시 (예: ETF를 이해하려면 주식 문서 먼저)

## 5. 기술

- [ ] MDX 문법 유효 (frontmatter YAML 파싱 에러 없음)
- [ ] `npm run reindex` 에러 없음
- [ ] 파일명 kebab-case
- [ ] 경로가 올바른 locale 디렉토리 (`ko/` 또는 `en/`)

## 6. 금지 확인

| 항목 | 확인 |
|------|:----:|
| "삼성전자", "테슬라" 등 개별 종목명 | ❌ 없어야 함 |
| "A증권사 추천" 등 특정 회사 추천 | ❌ 없어야 함 |
| "연 10% 보장" 등 수익 보장 표현 | ❌ 없어야 함 |
| 날짜 없는 수치 ("한도는 2천만원") | ❌ 없어야 함 |
| 블로그/커뮤니티 출처 | ❌ 없어야 함 |
| 면책 조항 누락 | ❌ 없어야 함 |
