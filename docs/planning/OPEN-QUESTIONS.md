## OQ-1. 레포/프로젝트 이름

현재 `docs-mcp`는 범용 엔진 이름이다. 금융 콘텐츠를 넣으면 이름을 바꿔야 하는가?

**선택지:**
- **A) `finance-docs-mcp`** — 도메인이 명확, MCP 디렉토리 검색에 유리 ✅
- B) `finance-mcp` — 짧고 간결
- C) `fin-docs` — 더 짧음, MCP 뉘앙스 없음
- D) `docs-mcp` 유지 — 범용 엔진 + 금융 콘텐츠 조합, 나중에 다른 도메인도 가능

**결정:** A — `finance-docs-mcp`
**근거:** 도메인(finance) + 역할(docs) + 기술(mcp)이 모두 이름에 드러남. MCP 디렉토리에서 검색 시 finance 키워드로 노출 유리.
**일자:** 2026-06-18

**후속 작업:** GitHub 레포 rename (`docs-mcp` → `finance-docs-mcp`), package.json name 변경

## OQ-2. 프론트엔드 배포

MCP + REST API는 Fly.io로 배포 확정. 웹 프론트엔드(검색 UI + 챗봇)는?

**선택지:**
- A) 자체 도메인 구매 (예: fin-docs.kr) + Vercel 배포

- C) 프론트엔드 없이 MCP 전용으로 시작, 나중에 추가

## OQ-3. 부동산 카테고리 범위

부동산은 금융/재테크의 큰 축이지만 깊이 들어가면 별도 프로젝트 수준.

**선택지:**
- **A) 기초만 (전세/월세 차이, 청약 개요, DSR/LTV 설명) — 5~8개 문서** ✅
- B) 중급까지 (청약 전략, 대출 비교, 세금 계산) — 15~20개 문서
- C) 제외 — 주식/금융상품에 집중
- D) Phase 2에서 별도 카테고리로 추가

**결정:** A — 기초만. 주식과 재테크(ISA, IRP 등)에 좁게 집중.
**근거:** 핵심 가치인 "투자/재테크 교육"에서 벗어나지 않으면서 부동산은 기초 개념만 커버. 깊이는 주식/금융상품/절세 계좌에 집중.
**일자:** 2026-06-18

## OQ-5. 커뮤니티 기여 시점

오픈소스로 공개하면 외부 기여를 받을 수 있지만, 초기에는 품질 관리가 어려움.

**선택지:**
- **A) 처음부터 CONTRIBUTING.md + PR 템플릿 준비** ✅
- B) Phase 1(MVP 50개) 완성 후 기여 가이드 공개
- C) Phase 2(100개) 이후, 품질 기준이 확립된 후

**결정:** A — 처음부터 준비. 단, 본질이 흐려져서는 안 됨.
**근거:** 오픈소스 프로젝트는 초기부터 기여 구조를 갖추는 게 커뮤니티 형성에 유리. 다만 품질 기준(정확성, 비추천 원칙, 출처 명시)을 CONTRIBUTING.md에 명확히 서술하여 본질(금융 교육 지식베이스)을 지킨다.
**일자:** 2026-06-18

**가드레일:**
- PR 리뷰 시 품질 체크리스트 필수
- 특정 종목/상품 추천 PR은 자동 거절
- 출처 없는 수치/제도 설명 PR은 수정 요청

## OQ-6. 법적 면책 조항

금융 콘텐츠는 "투자 조언"으로 오해받을 수 있다. 면책 조항을 어디에 어떻게 넣을지.

**선택지:**
- **A) 레포 README + 모든 MDX 문서 하단에 고정 문구** ✅
- B) README + 챗봇 시스템 프롬프트에만 (문서 본문에는 넣지 않음)
- C) 별도 DISCLAIMER.md + 챗봇 응답 시 자동 첨부
- D) 카테고리 허브(index.mdx)에만 한 번씩

**결정:** A — README + 모든 MDX 하단 고정 문구
**근거:** 법적 보호를 위해 콘텐츠가 노출되는 모든 곳에 면책 조항 필요. MDX 문서는 프론트엔드, 검색 결과, RAG 응답 등 다양한 경로로 소비되므로 문서 자체에 포함되어야 함.
**일자:** 2026-06-18

**면책 문구 초안:**
> 이 문서는 금융 교육 목적으로 작성되었으며, 투자 권유나 특정 금융상품 추천이 아닙니다. 투자 판단은 본인의 책임이며, 실제 투자 전 전문가와 상담하시기 바랍니다. 수치와 제도는 작성 시점 기준이며 변경될 수 있습니다.
